# FieldDescription
% Description of Names Files for Countries and Territories Format
%*************************************************************************

> Note:

   * The data format is tab-delimited text.
   * The names in the country files are sorted alphabetically and are
     displayed in Reading order (usually used in mapping applications; ex.
     Mount Everest)

 -------------------------------------------------------------------------

 FieldName        FieldDescription                          DataType  DataLength

 01. RC         =  Region Code                                    > tiny.int  1
                +  A code that determines the character
                +  mapping used in the Full_Name field
                +  (see  REGIONS.PDF for character mapping):
                +  1: Western Europe/Americas;
                +  2: Eastern Europe;
                +  3: Africa/Middle East;
                +  4: Central Asia;
                +  5: Asia/Pacific;
                +  6: Vietnam.

 02. UFI        =  Unique Feature Identifier                      > int       4
                +  A number which uniquely identifies the feature.

 03. UNI        =  Unique Name Identifier                         > int       4
                +  A number which uniquely identifies a name.

 04. LAT        =  Latitude of the Feature                        > N/A
                +  in ± decimal degree (WGS84):
                +        no sign (+): North;
                +  negative sign (-): South.

 05. LONG       =  Longitude of the Feature                       > N/A 
                +  in ± decimal degree (WGS84):
                +        no sign (+): East;
                +  negative sign (-): West.

 06. DMS_LAT    =  Latitude of the Feature                        > int       4
                +  in ± DegMinSec DDMMSS (WGS84):
                +        no sign (+): North;
                +  negative sign (-): South.

 07. DMS_LONG   =  Longitude of the Feature                       > int       4
                +  in ± DDMMSS (WGS84):
                +        no sign (+): East;
                +  negative sign (-): West.

 08. UTM        =  Universal Transverse Mercator                  > char      4
                +  coordinate grid reference.

 09. JOG        =  Joint Operations Graphic reference             > char      7

 10. FC         =  Feature Classification                         > char      1
                +  A: Administrative region;
                +  P: Populated place;
                +  V: Vegetation;
                +  L: Locality or area;
                +  U: Undersea;
                +  R: Streets, highways, roads, or railroad;
                +  T: Hypsographic;
                +  H: Hydrographic;
                +  S: Spot feature.

 11. DSG        =  Feature Designation Code                       > char      5
                +  A two to five-character code used to
                +  identify the type of feature a name is applied to.

 12. PC         =  Populated Place Classification                 > tiny.int  1
                +  A graduated numerical scale denoting
                +  the relative importance of a
                +  populated place.  The scale ranges
                +  from 1,  relatively high, to 5,
                +  relatively low.  The scale could
                +  also include NULL (no value) as a
                +  value for populated places with
                +  unknown or undetermined
                +  classification.

 13. CC1        =  Primary Country Code                           > char      2
                +  A two alphabetic character code uniquely
                +  identifying a geopolitical entity
                +  (countries, dependencies, and areas of special sovereignty).

 14. ADM1       =  First-order Administrative Division            > char      2
                +  A two alphanumeric character code uniquely
                +  identifying a primary administrative division of a
                +  country, such as a state in the United States.

 15. ADM2       =  Second-order Administrative Division           > var.char  255
                +  The name of a subdivision of a first-order
                +  admin. division, such as a county in the United States.

 16. DIM        =  Dimension
                +  Usually used to display elevation or population data.

 17. CC2        =  Secondary Country Code                         > char      2
                +  A two alphabetic character code uniquely
                +  identifying the country code of a particular name 
                +  if different than that of the feature.

 18. NT         =  Name Type                                      > char      1
                +  C: Conventional;
                +  D: Not verified;
                +  N: Native;
                +  V: Variant or alternate.

 19. LC         =  Language Code                                  > char      2
                +  A two alphabetic character code uniquely
                +  identifying a language of  a country
                +  if multiple official languages are used.

 20. SHORT_FORM =  A specific part of the name                    > var.char  128
                +  that could substitute for the full name.

 21. GENERIC    =  The descriptive part of the full name          > var.char  128
                +  (does not apply to populated place names).

 22. SORT_NAME  =  A form of the full name                        > var.char  200
                +  which allows for easy sorting of the name
                +  into alpha-numeric sequence. It is comprised
                +  of the specific name, generic name, and any articles
                +  or prepositions. This field is all upper case with
                +  spaces, diacritics, and hyphens removed and numbers are
                +  substituted with lower case alphabetic characters.

 23. FULL_NAME  =  The complete name                              > var.char  200
                +  which identifies the named feature.
                +  It is comprised of the specific name, generic name,
                +  and any articles or preposition
                +  (see to REGIONS.PDF for character mapping).

 24. FULL_NAME_ND =  Full name with Roman characters               > N/A
                  +  The  diacritics and special characters
                  +  are substituted with Roman characters
                  +  (see to REGIONS.PDF for character mapping).
                  +  ND = No Diacritics / Stripped Diacritics.

 25. MOD_DATE   =  The date a new feature was modified             > small     4
                +  added or any part of an existing feature was modified
                +  (YYYY-MM-DD).
